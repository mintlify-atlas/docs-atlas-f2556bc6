This guide walks you through a complete forward pass using the `chunk_gated_delta_rule` high-level API. You will prepare the required input tensors, call the kernel, and inspect the output and final recurrent state.

<Tip>
Use `bfloat16` or `float16` for all input tensors. FlashQLA does not support `float32` and will raise an error if you pass it.
</Tip>

## Running a forward pass

<Steps>
  <Step title="Import and prepare tensors">
    Import the function and create the five required input tensors. The example below uses a single sequence of length 4096 with 64 query/key heads, 64 value heads, and a head dimension of 128.

    ```python
    import torch
    from flash_qla import chunk_gated_delta_rule

    B, T, H_q, H_v, K = 1, 4096, 64, 64, 128
    dtype = torch.bfloat16
    device = "cuda"

    q    = torch.randn(B, T, H_q, K, dtype=dtype, device=device)
    k    = torch.randn(B, T, H_q, K, dtype=dtype, device=device)
    v    = torch.randn(B, T, H_v, K, dtype=dtype, device=device)
    g    = torch.nn.functional.logsigmoid(
               torch.randn(B, T, H_v, dtype=torch.float32, device=device)
           ) / 16
    beta = torch.randn(B, T, H_v, dtype=torch.float32, device=device).sigmoid()
    scale = K ** -0.5
    ```

    Tensor shapes at a glance:

    | Tensor | Shape | Dtype |
    |---|---|---|
    | `q` | `[B, T, H_q, K]` | `bfloat16` |
    | `k` | `[B, T, H_q, K]` | `bfloat16` |
    | `v` | `[B, T, H_v, K]` | `bfloat16` |
    | `g` | `[B, T, H_v]` | `float32` |
    | `beta` | `[B, T, H_v]` | `float32` |
  </Step>
  <Step title="Call the kernel">
    Pass the tensors to `chunk_gated_delta_rule`. Set `output_final_state=True` to also retrieve the recurrent state at the end of the sequence.

    ```python
    o, final_state = chunk_gated_delta_rule(
        q=q, k=k, v=v, g=g, beta=beta,
        scale=scale,
        output_final_state=True,
    )
    ```
  </Step>
  <Step title="Inspect the output">
    The function returns the output tensor `o` and the final recurrent state.

    ```python
    print(f"Output shape:      {o.shape}")           # [1, 4096, 64, 128]
    print(f"Final state shape: {final_state.shape}") # [1, 64, 128, 128]
    ```
  </Step>
</Steps>

## Complete example

```python
import torch
from flash_qla import chunk_gated_delta_rule

B, T, H_q, H_v, K = 1, 4096, 64, 64, 128
dtype = torch.bfloat16
device = "cuda"

q    = torch.randn(B, T, H_q, K, dtype=dtype, device=device)
k    = torch.randn(B, T, H_q, K, dtype=dtype, device=device)
v    = torch.randn(B, T, H_v, K, dtype=dtype, device=device)
g    = torch.nn.functional.logsigmoid(
           torch.randn(B, T, H_v, dtype=torch.float32, device=device)
       ) / 16
beta = torch.randn(B, T, H_v, dtype=torch.float32, device=device).sigmoid()
scale = K ** -0.5

o, final_state = chunk_gated_delta_rule(
    q=q, k=k, v=v, g=g, beta=beta,
    scale=scale,
    output_final_state=True,
)
print(f"Output shape:      {o.shape}")           # [1, 4096, 64, 128]
print(f"Final state shape: {final_state.shape}") # [1, 64, 128, 128]
```

## Using L2-normalized QK

Pass `use_qk_l2norm_in_kernel=True` to have the kernel apply L2 normalization to `q` and `k` before the attention computation. This is useful when your model trains with normalized query and key projections.

```python
o, final_state = chunk_gated_delta_rule(
    q=q, k=k, v=v, g=g, beta=beta,
    scale=scale,
    output_final_state=True,
    use_qk_l2norm_in_kernel=True,
)
```

## Stateful inference with `initial_state`

For autoregressive or multi-chunk inference, pass the recurrent state from one call as `initial_state` for the next. The state tensor has shape `[B, H_v, K, V]`.

```python
# First chunk
o1, state = chunk_gated_delta_rule(
    q=q, k=k, v=v, g=g, beta=beta,
    scale=scale,
    output_final_state=True,
)

# Second chunk — seed from the previous final state
o2, state = chunk_gated_delta_rule(
    q=q, k=k, v=v, g=g, beta=beta,
    scale=scale,
    initial_state=state,
    output_final_state=True,
)
```

For variable-length sequences packed into a single batch dimension, see the [variable-length sequences guide](/guides/variable-length-sequences).
