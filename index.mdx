FlashQLA is a high-performance linear attention kernel library built on [TileLang](https://github.com/tile-ai/tilelang), targeting NVIDIA Hopper (SM90) GPUs. It implements fused, warp-specialized CUDA kernels for the Gated Delta Rule (GDN) attention mechanism, achieving **2-3x forward** and **2x backward** speedup over the FLA Triton baseline across the Qwen3.5/Qwen3.6 model family.

## Why FlashQLA?

FlashQLA addresses the performance bottleneck in linear attention inference and training for large language models. The Qwen3.5 and Qwen3.6 model families use a Gated Delta Rule attention mechanism that existing Triton kernels handle inefficiently. FlashQLA closes this gap through three complementary optimizations.

<CardGroup cols={3}>
  <Card title="Auto Context Parallelism" icon="arrows-split-up-and-left">
    Gate-driven intra-card CP automatically splits long sequences under TP, improving SM utilization without manual configuration.
  </Card>
  <Card title="Algebraic Reformulation" icon="function">
    Hardware-friendly math reduces Tensor Core, CUDA Core, and SFU overhead without sacrificing numerical precision.
  </Card>
  <Card title="Fused Warp Kernels" icon="microchip">
    TileLang fused kernels with manual warpgroup specialization overlap data movement and computation.
  </Card>
</CardGroup>

## Hardware requirements

<Note>
FlashQLA currently requires NVIDIA Hopper architecture (SM90) or above. Ampere and earlier GPUs are not supported.
</Note>

| Requirement | Minimum version |
|---|---|
| GPU architecture | SM90 (NVIDIA H100, H200) |
| CUDA | 12.8 |
| PyTorch | 2.8 |
| Python | 3.10 |

## Get started

<Steps>
  <Step title="Install FlashQLA">
    Clone the repository and install with pip:
    ```bash
    git clone https://github.com/QwenLM/FlashQLA.git
    cd FlashQLA
    pip install -v .
    ```
  </Step>
  <Step title="Run your first forward pass">
    Import `chunk_gated_delta_rule` and call it with your query, key, value, gate, and beta tensors:
    ```python
    import torch
    from flash_qla import chunk_gated_delta_rule

    o, final_state = chunk_gated_delta_rule(
        q=q, k=k, v=v, g=g, beta=beta, scale=scale,
        output_final_state=True,
    )
    ```
  </Step>
  <Step title="Explore advanced features">
    Use variable-length sequences with `cu_seqlens`, access low-level forward/backward APIs, or run the benchmark suite to compare against FLA Triton and FlashInfer baselines.
  </Step>
</Steps>
